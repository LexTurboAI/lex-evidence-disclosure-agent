# Lex Turbo Agent Disclosure Standard

A lightweight audit standard for AI agents used in external communication.

## 10 principles

1. **AI status disclosure** — the recipient should know they interact with or receive content prepared by an AI system.
2. **Operator identity** — the responsible organization or operator should be identifiable.
3. **Scope of authority** — the agent should have a documented scope and should not exceed it.
4. **Human approval** — medium, high and critical-risk actions should require human review.
5. **Audit trail** — agent actions should preserve message content, timestamps, tool use, data access and approval status.
6. **Chain of custody** — logs should be chronological, complete and hashable for basic reconstruction.
7. **Human escalation** — high-risk matters should have a path to a human representative.
8. **Data minimization** — the agent should access only data needed for the task.
9. **Reserved activity guardrail** — the agent should not autonomously perform legal, medical, tax or other reserved professional activities.
10. **Complete disclosure notice** — a complete notice should include AI status, operator identity and human contact route.

This standard is used here as a deterministic demonstration framework. It is not legal advice.
